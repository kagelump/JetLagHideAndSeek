# T7 — Offline play-area setup (search + load from packs)

## Context

With T6's boundaries artifact installed, the app can set up a play area with
zero network: search boundary names locally, load the polygon from the pack.
This is the core payoff of the epic's motivation — Overpass flakiness can no
longer block starting a game.

Read first: `src/features/playArea/playAreaSearch.ts` (Photon search + the
result mapping), `src/features/map/playAreaBoundary.ts` /
`playAreaBoundaryConversion.ts` (what a loaded play area looks like,
including `maskHoles`), the AGENTS.md Play Area Rules (resolution order),
and T6's artifact format.

## What to build

### 1. Boundary store — `src/features/offline/boundaryStore.ts` (new)

Consumes the `boundaries` artifact installed by T5 (fill in T5's TODO):

- **Install-time split** (this means editing T5's code — open
  `installPack`'s per-kind `switch` in `src/features/offline/regionPacks.ts`
  and replace the `boundaries` TODO case): after the standard
  verify-and-write steps, `JSON.parse` the artifact **once**, then write
  two files into the pack dir — `boundaries-index.json`
  (`{ schemaVersion, regionId, levels, index }`) and
  `boundaries-polygons.json` (`{ polygons }`) — and delete the original
  combined file. A full parse here is fine and expected: install is a
  one-time foreground operation with a progress bar; the lazy-load rule
  protects _app start_, not install. No streaming parser needed.
- App start / post-install: parse only `boundaries-index.json` into memory
  (small), keyed by packId. `boundaries-polygons.json` is parsed lazily on
  the first polygon decode; decoded polygons go in an LRU (cap ~8).
- `searchBoundaries(query): BoundaryHit[]` — normalize the query with the
  same rules as the pipeline (port the normalizer; lowercase + NFKD + strip
  combining marks) and prefix/substring match against `normalized[]`.
  Rank: exact > prefix > substring, then lower `adminLevel` first, then
  larger `areaKm2`. Cap 20.
- `getBoundaryPolygon(packId, relationId)` — TypeScript port of T6's
  reference decoder (**length-prefixed rings** — see the resolution note in
  `docs/tasks/admin-boundaries-delta-encoding.md`), round-trip-tested
  against fixture vectors **generated by the pipeline encoder** (commit the
  fixture JSON under the test's `fixtures/`; regenerating it is a pipeline
  script flag).

### 2. Search integration

`usePlayAreaSearch` merges sources:

- Local results always compute (fast, synchronous over in-memory index) and
  render immediately, marked `source: "pack"`.
- Photon results append when the fetch succeeds (dedupe by OSM relation id —
  local wins). Photon failure/offline: local-only, no error state when
  local results exist.
- Result shape stays `PlayAreaSearchResult`; add the optional `source`
  field. The Play Area screen shows a subtle "offline" tag on pack results
  (reuse an existing badge style — don't invent new design tokens).

### 3. Loading integration

`loadPlayAreaByRelationId` resolution order gains packs (AGENTS.md order +
one step): bundled Tokyo/Osaka → memory cache → AsyncStorage cache →
**installed packs** → Overpass fetch.

The pack step: find the relation in any installed boundary index, decode
the polygon, and convert to the exact same play-area shape the Overpass
path produces — boundary FeatureCollection, bbox, center, label, and
`maskHoles` metadata. Route through the same conversion helpers
(`playAreaBoundaryConversion.ts`) the Overpass path uses so downstream code
can't tell the difference. Then write it into the AsyncStorage cache like a
fetched result so subsequent loads hit the cache.

Update AGENTS.md's Play Area Rules bullet with the new resolution order in
this PR (it documents the invariant).

### 4. Decode performance guard

Decoding a big multipolygon (a prefecture with islands) must not jank the
sheet. Measure on the Taiwan pack in a dev build; if decode >50 ms on
device, move decode off the press handler (InteractionManager.runAfterInteractions)
and show the existing loading state. Record the measured number in the PR
description (design.md flags this as an open risk).

## How to test

Jest:

- Decoder: fixture vectors from the pipeline encoder round-trip exactly.
- `searchBoundaries`: ranking (exact beats prefix beats substring; level
  ties broken by area), diacritics-insensitive query, CJK exact match, cap.
- Search merge: Photon mock rejects → pack results still render, no error;
  both succeed → dedupe by relation id with pack result first.
- `loadPlayAreaByRelationId`: relation present in a pack short-circuits
  before the (mocked, asserted-not-called) Overpass fetch; produces a play
  area deep-equal in shape to the Overpass-path fixture for the same
  relation; populates the AsyncStorage cache.
- Eviction: removing the pack (T5 `removePack`) drops index + polygons; a
  play area already loaded from it stays loaded (it lives in the play-area
  cache now — assert that).

Manual (M2 exit criterion): device with NL + Taiwan packs, full app reset,
airplane mode ON from launch: search "utrecht" → select → play area renders
with mask; repeat with "台北" on the Taiwan pack.
