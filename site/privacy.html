Privacy Policy

Effective: June 5, 2026

Hide & Seek Mapper (“the App”) is a local-first mobile tool for planning and sharing setup for hide-and-seek games. This policy explains what data the App handles and how.

1. No Accounts or Sign-Up

The App does not require an account, does not ask you to sign in, and does not collect or transmit any personal identifiers. There is no user registration of any kind.

2. No Analytics, No Crash Reporting, No Telemetry

The App does not include any analytics SDKs, crash-reporting services, or telemetry frameworks. It does not track how you use the App, how often you use it, or what features you interact with. No usage data of any kind is collected or transmitted.

3. Data Stored on Your Device

All game data lives entirely on your device in local storage (AsyncStorage and an on-device file cache). This includes:

  • Your play-area choice (an OpenStreetMap relation ID and its label and boundary).
  • Hiding-zone settings (radius in meters and selected transit presets).
  • Questions and their answers (radar distances, matching candidates, answer states).
  • Cached OpenStreetMap map data for faster offline searches.

None of this data is uploaded to any server. You can clear it at any time by deleting the App from your device.

4. Location Data

The App requests foreground-only location permission for two purposes:

  • To show your current position on the map so you can see where you are relative to the play area.
  • To set the center point when you create a radar or matching question.

Your location is never stored in local storage, never transmitted to any external service (except as coordinates in the anonymized network requests described below), and never shared with third parties. The App does not collect or record your location history.

5. Network Requests

The App makes network requests to the following services. None of these requests include a persistent device identifier, account ID, or personal information:

5.1. Photon Geocoding (photon.komoot.io)

When you search for a play area by name, the App sends your search query text to the Photon geocoding API. No other data is included in the request.

5.2. Overpass API (overpass-api.de)

When you select a play area or create a matching question, the App queries the Overpass API for OpenStreetMap data (boundary geometry and nearby points of interest). The request includes the OSM relation ID, or a latitude/ longitude pair and a search radius — only enough to describe the area of the map you want to query. No persistent identifier or personal data is attached.

5.3. Offline Pack Manifest

The App may fetch a manifest of available offline map-data packs from a content-delivery network. This request contains no user data.

6. Game-State Sharing

The App includes a feature that lets you share your game setup (play area, hiding zones, and questions) with another player by generating a link or QR code. Sharing is always user-initiated — the App never transmits game state automatically.

The shared data is encoded directly in the link itself. No copy of the link or its contents is stored on any server. A randomly generated game ID is included in the link only to help the recipient’s device recognize a new game setup; it is not tied to any identity.

7. Third-Party Map Data

The App ships with bundled map data from OpenStreetMap (© OpenStreetMap contributors, licensed under the Open Database License) and ODPT public transit data. This data is embedded in the app bundle; loading it does not involve any network request and does not transmit any information about you. Attribution for these data sources is available in the App’s Settings screen.

8. Children’s Privacy

The App does not knowingly collect, store, or share personal information from anyone, including children under the age of 13. Because the App collects no personal data at all, there is nothing to age-gate.

9. Changes to This Policy

If this privacy policy changes, the updated version will be posted at this same URL with a new effective date. Because the App does not collect contact information, we cannot notify you individually of changes.

10. Contact

If you have questions about this privacy policy, you can open an issue on the project’s GitHub repository.
